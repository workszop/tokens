How it works

  1. Start with individual characters as tokens
  2. Find the most frequent pair of adjacent tokens
  3. Merge that pair into a new token
  4. Repeat until reaching the target vocabulary size

Technical Note

This visualization omits the end-of-word marker (typically denoted as </w>) for clarity. In real BPE, each word's final character is marked with a special symbol, allowing the algorithm to distinguish tokens like "er" at the end of a word ("lower") from "er" in the middle ("merry").

This ensures BPE knows not to merge tokens across word boundaries. In this app, we use a | separator between words that serves a similar role but is hidden from display.
